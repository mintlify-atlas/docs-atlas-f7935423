How is a skill for AI coding agents that turns any "how does X work?" question into a clear, senior-engineer-quality architectural explanation. It automatically decides whether a question needs a single-agent pass or a fan-out to parallel explorer agents — then synthesizes everything into one coherent picture. For deeper analysis, an optional critique mode spawns independent critics across multiple AI models to surface architectural problems.

## What How does

<Steps>
  <Step title="You ask a question">
    Type `/how` followed by any architecture question — a subsystem, a feature flow, an architectural overview, or a runtime trace.
  </Step>
  <Step title="How assesses complexity">
    Simple questions (a single module, a narrow function) run end-to-end in one agent. Complex questions (cross-cutting subsystems, multi-service flows) fan out to 2–4 parallel explorer agents.
  </Step>
  <Step title="Explorers gather findings">
    Each explorer agent traces a different slice of the codebase — data models, rendering pipelines, service boundaries — reading actual code, not guessing from file names.
  </Step>
  <Step title="You receive a structured explanation">
    A synthesis agent reconciles the explorers' findings into one explanation: Overview, Key Concepts, How It Works, Where Things Live, and Gotchas. For critique mode, independent critics then review the architecture across multiple models.
  </Step>
</Steps>
